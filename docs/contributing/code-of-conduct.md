# 行動規範

ArchiTrackプロジェクトでは、すべてのコントリビューターが尊重され、包括的な環境を提供することを目指しています。

---

## 私たちの約束

オープンで誰もが参加しやすいコミュニティを育てるため、コントリビューター・メンテナーとして、以下を約束します：

- 年齢、体の大きさ、障碍、民族性、性同一性と性表現、経験レベル、国籍、個人の見かけ、人種、宗教、性的同一性と性的指向に関わりなく、すべての人にとってハラスメントのない参加体験を提供します

---

## 私たちの基準

プロジェクトコミュニティを作り上げる上での行動例：

### ポジティブな行動

- **尊重**: 異なる視点や経験を尊重する
- **建設的**: フィードバックは建設的に行い、受け入れる
- **協力的**: チームワークを重視し、互いに助け合う
- **包括的**: すべての人が参加しやすい環境を作る
- **感謝**: 他者の貢献に感謝を示す
- **プロフェッショナル**: 技術的な議論に集中する

### 許容されない行動

- **ハラスメント**: 性的な言葉や画像の使用、または性的な関心や誘いかけ
- **中傷**: 他者を侮辱的または軽蔑的に扱う
- **攻撃**: 個人的または政治的な攻撃
- **プライバシー侵害**: 許可なく他者の個人情報を公開する
- **不適切な行為**: プロフェッショナルな環境で不適切とされるその他の行為

---

## 私たちの責任

プロジェクトメンテナーは、許容される行動の基準を明確にする責任があり、何らかの許容されない行動に対して適切かつ公平な是正措置を取ることが期待されています。

プロジェクトメンテナーは、この行動規範に沿っていないコメント、コミット、コード、wikiの編集、issue、その他の投稿を削除、編集、または拒否する権利と責任を有しています。また、不適切、脅迫的、攻撃的、または有害と判断される行動を取ったコントリビューターを一時的または恒久的に追放する権利と責任を有しています。

---

## 適用範囲

この行動規範は、個人がプロジェクトやそのコミュニティを代表している場合、プロジェクトの場内外の両方で適用されます。

プロジェクトやコミュニティを代表することの例には、以下が含まれます：

- プロジェクトの公式メールアドレスの使用
- 公式ソーシャルメディアアカウントからの投稿
- オンラインまたはオフラインイベントでの代表としての行動

---

## 報告

ハラスメント、虐待、またはその他の許容されない行動の事例は、プロジェクトチームに報告してください：

- **メール**: security@example.com
- **GitHub**: プライベートメッセージ

すべての報告は機密に扱われ、調査されます。プロジェクトチームは、報告者のプライバシーとセキュリティを維持する義務があります。

---

## 執行

行動規範に従わない場合、または違反者に対する執行措置は、プロジェクトチームが適切と判断した方法で決定されます。

**執行段階:**

1. **警告**: 非公開で警告を発する
2. **一時的な追放**: 一定期間、プロジェクトへの参加を禁止
3. **恒久的な追放**: プロジェクトから恒久的に追放

---

## コミュニティでの相互作用

### Issue・PRでのコミュニケーション

**良い例:**

```markdown
この実装について質問があります。
〇〇の部分で△△の方法を検討されましたか？
理由があれば教えていただけると助かります。
```

**悪い例:**

```markdown
この実装は間違っている。
〇〇をするべきだ。
```

### コードレビュー

**良い例:**

```markdown
このロジックは素晴らしいですね！
ただ、エッジケースとして〇〇の場合を
考慮する必要があるかもしれません。
```

**悪い例:**

```markdown
このコードは読みにくい。
全部書き直すべき。
```
